# 2025-08-16

一、核心技术
 
（一）区块链
 
- 定义：一种分布式数据库，由按时间顺序排列的数据区块以链条方式组合而成，并通过密码学保证其不可篡改和伪造。
- 关键特性：包括去中心化、共识机制、加密技术等。共识机制除了PoW、PoS，还有DPoS（委托权益证明），代币持有者投票选出代表来验证交易和创建区块；PBFT（实用拜占庭容错），通过多轮投票达成共识，适用于已知验证者身份的系统。
- 分类：公有链如比特币、以太坊；联盟链如Hyperledger Fabric；私有链由单一实体控制。
 
（二）智能合约
 
- 定义：自动执行合约条款的计算机程序，运行在区块链上，无需第三方信任机构介入。
- 特点：不可篡改、透明公开、自动执行。
- 编程语言：Solidity、Vyper（以太坊）、Rust（其他区块链平台）等。
 
（三）去中心化应用（DApps）
 
- 定义：运行在区块链上的应用程序，具有去中心化、数据透明和安全性高等特点。
- 结构：前端用户界面（HTML/CSS/JavaScript）、后端逻辑（智能合约）、区块链网络（如以太坊、EOS等）。
- 应用场景：除了金融、游戏、社交、供应链管理，还有DeFi（去中心化金融），如Uniswap交易平台、Aave借贷平台；NFT（非同质化代币），如OpenSea、Rarible等平台；DAO（去中心化自治组织），如MolochDAO。
 
二、数字货币与钱包
 
- 数字货币：如比特币、以太坊、USDT等，是区块链上的数字资产，具有去中心化、匿名性（部分）、可追溯等特点。
- 钱包：用于存储、发送和接收数字货币的工具，分为热钱包（在线钱包）和冷钱包（离线钱包），私钥管理至关重要。
 
三、跨链技术
 
- 定义：实现不同区块链网络之间的互操作性，使资产和数据能在不同链之间自由流动。
- 技术类型：原子交换、侧链、中继链、桥接等。
- 面临挑战：安全性、性能、兼容性等。
 
四、Web3生态与工具
 
- 开发工具：Truffle、Hardhat（以太坊开发框架），Remix（在线IDE），Ganache（本地测试网络）等。
- 浏览器：Etherscan、Blockchain.com、CryptoCompare等，用于查看区块链上的交易、地址和智能合约信息。
- 钱包应用：MetaMask（浏览器插件钱包）、Trust Wallet（移动钱包）等。
 
五、Web3相关概念
 
- 去中心化身份（DID）：用户的身份数据被加密并分散存储在区块链网络上，用户可以自主管理其身份证明和相关数据，使用私钥进行安全验证，ENS是知名项目。
- SocialFi：将社交互动与资产生成、交换和增值相结合，用户在参与社交活动时可得到经济奖励，如代币或NFT，同时保证对自己数据的所有权和控制权，代表平台有Mirror。
- GameFi：结合了加密货币、NFT和DeFi，玩家的时间和技能可以转化为现实世界的财富，Axie Infinity是典型代表。

# 2025-08-13

以太坊测试网络学习笔记
 
一、质押模式对比（Pooled Staking Vs Solo Staking）
 
（一）Pooled Staking（质押池）
 
参与方式：模式简洁，用户仅需质押自身ETH  。
资金特性：不锁死资金流动性，且质押衍生品（LST）可产生额外收益 。
风险点：用户不直接掌控资金，若质押池出现安全风险，会对以太坊网络构成较大威胁 。
 
（二）Solo Staking（独立质押）
 
准入条件：需 32 ETH 的准入资金，资金处于锁定状态，流动性差 。
运维要求：需自行维护节点安全，运维节点技术门槛高，还需承担服务器等硬件成本（如以太坊节点服务器的部署、维护开支 ） 。
 
二、节点多样性
 
（一）客户端分类
 
以太坊节点客户端主要分共识客户端（Consensus Clients）与执行客户端（Execution Clients） 。常见共识客户端有 Lighthouse、Nimbus 等；执行客户端有 Geth、Besu 等 。
 
（二）多样性价值
 
以太坊客户端具备多样性，共识层和执行层存在多种规范实现。共识层客户端与执行层客户端可灵活组合，丰富网络生态 。
伴随生态发展，节点采用不同客户端的比例逐渐合理化，降低单一客户端依赖风险，提升网络健壮性与抗攻击性 。
 
三、运行节点的要素
 
以太坊网络的稳定性依托节点运行，关键要素包括：
 
可运维性：节点需具备稳定运行能力，且便于技术人员进行日常维护、更新、故障排查等操作 。
可观测性：能够对节点的运行状态（如区块同步进度、交易处理速度、资源占用情况等 ）进行监测，及时发现性能瓶颈或异常问题 。
安全性：从网络防护（抵御 DDoS 攻击等 ）、数据安全（保障节点数据不被篡改、窃取 ）等维度，确保节点安全，维护以太坊网络整体安全 。
 
四、以太坊架构深度解析
 
（一）架构升级背景与核心变化
 
The Merge 升级是以太坊关键变革，共识机制从工作量证明（PoW）切换为权益证明（PoS）。在 PoS 架构下，一个完整的以太坊节点由执行层客户端（EL）、共识层客户端（CL）、验证者客户端（Validator）三部分构成 ，协同保障网络运行 。
 
（二）各层级组件与交互逻辑
 
共识层（Consensus layer）：承担共识达成职责，包含 Blocks（区块数据 ）、P2P（节点间点对点通信 ）、RANDAO（随机数生成，用于共识过程 ） 、Fork choice（分叉选择，确定合法链 ）等组件。通过 Engine API 与执行层进行交互，传递共识相关指令与数据 。
执行层（Execution layer）：聚焦交易与状态处理，涵盖 P2P（支持节点交互 ）、State（data）（维护区块链状态数据 ）、TXs（mempool）（交易内存池，暂存未上链交易 ）、EVM（以太坊虚拟机，执行智能合约 ） 。借助 JSON - RPC API 与用户或 WEB3 应用对接，接收交易请求、反馈执行结果 。
验证者（Validators）：作为网络共识的参与主体，通过 Beacon APIs 接入共识层，参与区块提议、验证等流程，质押 ETH 保障网络安全，获取相应奖励，同时因作恶会受惩罚 。
 
五、容器化 & 可观测性实践
 
（一）容器化技术驱动部署变革
 
在云技术发展浪潮中，软件部署模式不断演进。容器化技术（典型代表如 Docker 实现应用打包隔离，Kubernetes 负责容器编排管理 ）成为关键支撑：
 
 标准化与降本：让软件部署流程标准化，降低环境适配等成本，不同团队、环境间，可快速复用部署配置 。
 环境屏蔽与稳定：屏蔽底层环境差异（如不同操作系统、服务器配置 ），确保应用在开发、测试、生产环境表现一致，使部署流程更稳定、可预期 。
 
（二）可观测性赋能节点管理
 
可观测性是保障以太坊节点乃至网络稳定的重要手段：
 
 量化与监测：通过各类工具（如 Prometheus 采集 metrics 数据、Grafana 可视化展示 ），对节点的资源占用（CPU、内存、磁盘 I/O 等 ）、运行状态（区块同步高度、交易处理延迟 ）等信息进行量化采集与实时监测 。
 问题响应与预防：第一时间发现节点异常（如区块同步卡顿、交易堆积 ），助力技术人员快速定位、解决问题；同时，基于历史数据与趋势分析，可对潜在风险（如资源不足引发的节点故障 ）进行预判、预防 ，保障以太坊网络持续稳定运行 。

# 2025-08-12

DeFi 学习笔记
 
一、核心产品介绍
 
（一）Uniswap
 
以太坊上最大的去中心化交易所（DEX），通过自动化做市商（AMM）模型实现无需订单簿的代币交易，各版本迭代：
 
V1（2018）：AMM 机制，支持 ETH 和 ERC - 20 对
V2（2020）：支持任意 ERC20/ERC20 交易对，新增闪电贷、价格预言机
V3（2021）：推出集中流动性（LP 定制价格区间）
V4（2025）：引入可定制化流动性池（Hooks 机制）
 
（二）MakerDAO & DAI
 
MakerDAO：以太坊上首个去中心化稳定币协议，主打超额抵押借贷生成稳定币 DAI，发展历程：
单抵押 DAI（Sai，2017）：仅支持 ETH 作为抵押资产
多抵押 DAI（MCD，2019）：支持多种抵押品（WBTC、USDC 等）
Maker Endgame（2023 - 2024，重大改革）：涉及子 DAO、算法稳定币、RWA 扩展
DAI：去中心化稳定币，由 MakerDAO 发行，1 DAI≈1 美元，采用超额抵押机制（用户存入加密资产如 ETH、WBTC 作抵押品借出 DAI ，需存入＞100%抵押品，如 150% ），完全链上运行，无中心化机构控制，由智能合约和 MKR 持有者治理
 
（三）SushiSwap
 
最初是 Uniswap V2 的分叉，后发展出自身生态的去中心化交易所（DEX），采用自动化做市商（AMM）模型 。2020 年 9 月开展“吸血鬼攻击”：分叉 Uniswap V2 ，推 SUSHI 代币激励吸引 Uniswap LP 迁移（用户提供 Uniswap LP 凭证可挖矿获 SUSHI ，成功后抽走 Uniswap 流动性至 Sushi ），短期 TVL 超 Uniswap ，但创始人抛售 SUSHI 引发信任危机，Uniswap 反击发布 UNI 代币空投
 
二、产品启示与策略
 
（一）可组合性（Composability）
 
DeFi 乐高特性，协议间可无缝集成（如 Uniswap + Aave ），助力创新加速（开发者无需重复造轮子，直接调用现有协议 ）。案例：SushiSwap 分叉 Uniswap ，并扩展借贷（Kashi ）和质押（xSUSHI ）
 
（二）吸血鬼攻击（Vampire Attack）
 
竞争策略，通过代币激励短期掠夺对手流动性，但存在不可持续、依赖代币通胀风险（如 SUSHI 创始人抛售 ）。案例：SushiSwap 2020 年抽干 Uniswap 流动性
 
（三）代币标准化（Token Standards）
 
ERC - 20 普及（如 Uniswap V2 支持所有 ERC - 20 交易对 ）推动行业互通，降低开发者接入生态门槛（只需遵循标准 ）。案例：DAI（ERC - 20 ）成为 DeFi 基础货币
 
（四）治理与团队稳定性
 
去中心化≠无管理，需平衡治理与团队稳定性。案例：Uniswap 靠专业团队，SushiSwap 因社区混乱、创始人信任危机（如 Chef Nomi 抛售 SUSHI ）受影响
 
（五）长期主义 vs 短期投机
 
Uniswap 专注 AMM 迭代（V3→V4 ），稳居龙头；SushiSwap 盲目扩张、产品分散，最终边缘化
 
三、DAI 与 UST 对比
 
（一）稳定机制
 
DAI采用超额抵押模式，依靠ETH、WBTC等真实加密资产抵押，保障价值稳定；UST则通过算法调控，依赖LUNA代币销毁/铸造维持价格，无实际资产兜底。
 
（二）抵押品与去中心化程度
 
DAI抵押品为加密资产 + 潜在RWA（现实世界资产），依托MKR治理与智能合约，实现完全去中心化；UST无抵押品，依赖Terra基金会，存在部分中心化隐患。
 
（三）清算与风险应对
 
DAI设置自动拍卖抵押品机制，抵押率下跌时及时止损，抵御市场波动；UST无有效清算流程，依赖市场套利“自我修复”，遭遇信心危机时，易陷入“死亡螺旋”（抛售→脱锚→崩盘 ）。
 
（四）实际表现验证
 
DAI在2020年市场危机中保持锚定，经受住熊市考验；UST于2022年5月崩盘，LUNA归零、UST脱锚，凸显超额抵押模型（DAI）因“真实资产支撑 + 清算机制止损”，在稳定性上更具优势。

总结：超额抵押的 DAI 更可靠，因有真实资产支撑（不依赖市场情绪 ）、具备清算机制（避免系统性坏账 ）

6. 多任务优先级辨别力：合理安排工作顺序，保障高效推进。
 
二、Web3运营工作：魅力与风险
 
（一）魅力
 
1. 去中心化&远程（Decentralize & Remote）：工作模式灵活，突破地域限制。
2. 自主权（Ownership）：对项目有更多主导、决策空间。
3. 无需许可&去信任（Permissionless & Trustless）：参与门槛相对开放，基于规则协作。
4. 激励机制（Incentive）：有丰富激励，如代币、社区荣誉等。
5. 社区驱动（Community - driven）：运营与社区深度绑定，共创价值。
 
（二）风险
 
1. 高风险：行业尚在发展，项目成败、市场波动影响大。
2. 野蛮生长：行业规范待完善，部分项目运作不规范。
3. 灰色地带（合规）：业务模式可能涉及合规模糊区域。
4. 无实体公司/境外公司：合作、权益保障等存在不确定性。
5. 处处是坑：行业新、变化快，易踩业务、合作等方面的“雷” 。

三、新手运营难点
 
1. 行业基础知识匮乏，对Web3生态、规则等不熟。
2. 语言障碍，涉及国际化社区、技术术语等沟通难。
3. 信息过载且难判重点，不知聚焦关键内容。
4. 把握不住工作优先级，任务推进混乱。
5. 远程协作困难，沟通、进度同步易出问题。
6. 项目周期短，需快速学习、出成果，压力大。
 
四、给新手运营的建议

Web3技术核心要点
 
1. 以太坊生态剖析：以太坊是Web3的关键公链，其Layer 1是基础层，处理交易和智能合约执行，Layer 2则是为解决Layer 1的性能瓶颈而存在的扩展方案，如Optimistic Rollups、zk-Rollups等。以太坊2.0致力于通过信标链实现权益证明（PoS）共识机制，提升安全性和可扩展性，分片技术将大区块链拆分为多个分片并行处理交易。以太坊虚拟机（EVM）能让智能合约在以太坊网络上运行，Gas机制用于衡量执行智能合约所需的计算资源和费用，账户模型包含外部拥有账户（EOA）和合约账户，二者在交易发起、存储和逻辑执行上有明显区别 。
2. 智能合约开发进阶：在开发语言上，Solidity除基础语法外，像修饰符可限制函数调用条件，事件用于合约与外部交互时的通知。开发工具中，Remix IDE便于在线开发和测试；Truffle和Hardhat功能更全面，能管理项目、部署合约和进行测试，还支持插件扩展。部署和调用智能合约时，交易涉及签名、广播等流程，Gas需合理设置，应用二进制接口（ABI）用于定义合约与外部交互方式，Web3.js和Ethers.js库可实现与合约交互。智能合约安全方面，常见漏洞有重入漏洞、整数溢出等，可通过遵循安全开发规范、使用安全库和严格审计来防范。
3. 去中心化应用（DApp）开发：前端与区块链结合时，HTML、CSS、JavaScript构建用户界面，通过Web3.js或Ethers.js与钱包交互，实现连接钱包、发送交易等功能。去中心化存储如IPFS，采用内容寻址，数据存储在众多节点上，具有抗审查、高可用等优势，开发者可利用其存储DApp相关数据。去中心化身份（DID）让用户掌控自己身份信息，基于区块链的加密技术实现身份验证和管理，解决传统身份系统痛点 。在高级应用开发中，DeFi涵盖借贷、交易、保险等领域；NFT应用于数字资产确权和交易；DAO通过智能合约实现组织治理，成员通过持有治理代币进行投票决策。
 
Web3行业发展趋势
 
1. 技术融合创新：Web3与物联网结合，可让物联网设备数据上链，实现数据确权和安全共享，如供应链溯源场景；与人工智能融合，AI可分析链上数据提供决策支持，区块链保障AI模型数据的安全性和隐私性 。零知识证明技术逐渐成熟，在保护用户隐私前提下验证交易和数据，如隐私币项目和匿名交易平台应用。
2. 市场与监管动态：随着Web3应用普及，市场规模持续扩大，吸引大量资金流入，NFT市场的火爆和DeFi锁仓量增长就是体现。监管方面，各国对Web3的态度差异较大，部分国家积极探索监管框架，促进创新发展，部分国家则严格限制甚至禁止，未来全球监管协调将是行业面临的重要课题。

# 2025-08-07

Web3打工人风险控制学习笔记
 
一、分享背景与主体
 
由上海曼昆律师事务所饶炜彤律师分享，聚焦Web3打工人面临的法律风险及应对，助力从业者合法合规开展工作 。
 
二、Web3常见罪名
 
Web3行业存在诸多法律风险，对应常见罪名有：集资诈骗、非法吸收公众存款、组织/领导传销活动、开设赌场、非法经营 ，这些是从业者需警惕的“雷区”。
 
三、不同岗位风险差异
 
核心管理层：处于决策核心，对项目整体运作负责，责任重，风险高，一旦项目涉法，易被追究责任。
宣传/社区负责人：负责推广、运营社区，若宣传内容违规（如虚假宣传、诱导投资等 ），或社区涉传销、非法募资，会牵连自身。
核心链条人员：参与项目关键环节，如业务流程、资金流转等，对项目运作知之甚多，风险伴随关键工作而来。
纯粹技术提供者：并非“法外之地”，若明知项目可疑却不询问、不防范，会因“知情推断”被追责，免责前提是尽力防范、保持审慎。
 
四、Web3招聘与入行考量
 
招聘信息多样，如技术大佬（密码学、Java后端等岗位 ）、媒体编辑等，但入行需谨慎：
 
关注项目是否在明确、受监管司法辖区注册，有无合规审计/测试，是否有KYC（用户身份识别 ）、AML（反洗钱 ）制度 。
查看项目是否公开负责人、团队背景、资金来源，有无高风险模块、扩张方式，以此判断项目合法性与稳定性。
 
五、重点风险领域
 
（一）资金汇集
 
涉及交易所、虚拟货币基金时，需关注：是否涉法币交易，海外合规（AML、KYC ）是否完备，技术上是否排除中国大陆用户（规避国内监管风险 ） 。
 
（二）虚拟货币发行与募资
 
虚拟货币发行和募资（如ICO ）是红线，易涉集资诈骗（如光锥币、大圣币案例 ），海外持牌复杂昂贵，从业者需远离这类违规行为。
 
（三）传销与赌场风险
 
项目扩张要合规，避免多级返佣、团队奖励等传销模式；合约交易等非典型场景也可能涉赌，需警惕。
NFT玩法设计要合规，不能变相搞传销、赌博或违规募资。
 
（四）洗钱与外汇管制
 
Web3业务可能成为洗钱链条一环或外汇兑换媒介，非管理层难知晓资金全貌，遇到不清楚情况，要提高警惕，避免卷入洗钱、外汇违规案件 。
 
综上，Web3打工人需清晰岗位风险，选项目、做工作时，从合规注册、信息公开、风险模块等多维度判断，远离红线行为，保护自身合法权益 。

web3故事会

一、核心人物与项目

（一）Vitalik与以太坊

以太坊：区块链2.0代表，支持智能合约，为众多DApp提供开发平台。

贡献：开创智能合约时代，推动区块链从单一货币向多元应用发展。

（二）BM与EOS

比特股（BTS）：2014年推出，首个Dex，TPS达3000+，采用DPOS治理机制，有稳定币、资产承兑等创新点，但最终失败。

EOS：定位区块链3.0，号称百万TPS（实际数千），采用DPOS制度（21个超级节点），免除Gas费优化用户体验，后因过度中心化、资源模型失败等问题逐渐沉寂。
 
二、关键概念与事件

（一）ICO（首次代币发行）

定义：区块链行业创新融资方式，项目方以加密货币换取代币/硬币。

发展：2016年起大量涌现，帮助项目快速获资、启动社区与营销推广，相比传统IPO监管少、跨地域、更迅速。

风险：在大陆属敏感词，存在监管风险与投资风险。

（二）DApp（去中心化应用）

爆发：2018年为DApp爆发年，年末共1423个DApp，其中以太坊1045个、EOS235个、TRON97个、STEEM46个。

市场分布：游戏类占比最大，其次是博彩、去中心化交易所、资金盘等。
 
三、区块链治理

（一）重要性

关乎区块链项目的发展方向、决策效率与社区信任，良好治理是链存活的关键因素之一。

（二）以太坊与EOS治理对比

以太坊：通过社区共治、改进提案等方式逐步完善，注重去中心化与技术演进。

EOS：因DPOS治理危机（过度中心化）、社区分裂等问题导致治理失败，影响项目发展。
 
四、行业发展阶段

（一）2014年：启蒙阶段
BTS与ETH开启区块链应用新篇，BTS作为首个Dex引领去中心化交易探索。

（二）2016-2017年：ICO阶段
EOS等项目借ICO融资快速发展，行业热情高涨但也潜藏风险。

（三）2018年：DApp爆发与治理生态之争

DApp数量激增，各公链在治理模式、生态建设上竞争激烈，暴露诸多问题。

（四）2019年至今：反思与沉淀
对EOS等项目失败原因深入分析，行业回归理性，探索更可持续的发展路径。
 
五、总结
 
区块链行业历经起伏，从ICO火爆到DApp爆发，再到如今的反思沉淀，不断在技术、治理和生态等方面探索。未来，随着技术融合、监管完善及生态建设推进，区块链有望拓展应用，但需平衡去中心化与效率、创新与合规。

Web3 钓鱼攻击

一、常见攻击场景
1. Google搜索广告钓鱼：攻击者伪装成Aave等知名项目投虚假广告，诱用户点钓鱼网站。
2. 假Zoom会议钓鱼：通过Telegram发假Zoom邀请链接，用户点击或失资产。

二、攻击基础模式
1. 建立信任：伪装专家/客服骗信任。
2. 制造状况：谎称钱包有风险引恐慌。
3. 诱导操作：给假工具/网站诱输私钥。
4. 骗取资产：获私钥后转走资产。

三、Web3与Web2钓鱼差异
1. 目标：Web2偷账号，Web3偷资产。
2. 恢复：Web2可申诉，Web3难追回。
3. Web3风险：一次交易或授永久权限；钱包难锁号；部分交易藏转账/授权。

四、反钓鱼训练
Unphishable平台模拟真实攻击。

五、防范建议
1. 慎点非官方链接。
2. 不泄露私钥/助记词。
3. 用知名安全钱包。
4. 仔细核对网站域名。
5. 多学安全知识。

六、总结
钓鱼易实施危害大，通过训练和案例分享，可提升用户安全认知，防范攻击。
