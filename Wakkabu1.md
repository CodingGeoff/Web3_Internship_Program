# 2025-08-17

今日學習主題：Web3線上＆線下活動規劃差異與注意要點

在規劃 Web3 活動時，線上（Virtual）與線下（Physical）在目標、流程與風控上有明顯差異：線上強調「可擴展性與即時互動」，適合全球聽眾、降成本與內容長尾化；線下則以「沉浸式體驗與信任建構」為主，利於深度交流、實物示範與媒體曝光。策劃要點上，線上需把技術與用戶路徑放在首位——確保音訊/視訊穩定、回放錄製機制、聊天室與問答治理、以及防刷與假帳號策略；若有空投或白名單，須設計好防刷、KYC 與發放規則。線下則重視場地網路、現場安全、硬體錢包展示與講者彩排，以及物流、攝影與法務同意書。兩者在合規面共同需注意：公開合約地址、風險揭示、獎勵發放的稅務與 AML 要求。互動設計亦不同——線上以 Poll、Q&A、即時空投提升參與度；線下偏向工作坊、小組討論與實操示範來提高轉化。衡量指標方面，線上看 peak listeners、回放次數、互動率與社群新增；線下關注到場率、名片交換數、媒體引用與深度回饋。內容沉澱是兩種形式都必須做好的環節：線上直接剪輯短片與章節化回放；線下需快速整理講義、QA 與活動精華上傳。最後建議：無論線上或線下，都要先做完整彩排、備援方案與透明公告，並把安全教育（勿簽可疑合約、核實官方連結）內建於活動流程，才能把一次性流量有效轉化為長期社群資產。

# 2025-08-16

在 Web3 領域，薪酬形態比傳統更為多元：
1. 法幣薪資（銀行／國際匯款）
2. 穩定幣支付（USDC/USDT）
3. 代幣/空投（token grants）
4. 混合型薪酬（法幣＋代幣＋績效獎勵）。
代幣薪酬通常會搭配 vesting 與 cliff 條款以綁定貢獻者；穩定幣便利跨境但需注意匯率、稅務申報與對方合規性。實務上建議：薪資發放應明確貨幣種類、兌換機制、發放週期與手續費承擔方，並採用受信託或第三方支付服務降低拖欠風險。

合同與維權要點：書面合約是核心，必列主要條款—工作職責、薪酬組成（含代幣分配與 vesting 排程）、IP 歸屬、保密條款、解除條件、違約金、爭議解決方式（仲裁/法院）、適用法域與稅務責任。代幣項目特別要加上「代幣估值波動風險揭示」與「鎖倉/回收（clawback）」條款。若遇糾紛，應保存溝通紀錄、工時/任務證據與鏈上交易憑證，必要時委任律師或透過仲裁機制處理；跨境情況下，選擇明確的管轄法律能降低後續執行成本。

零經驗轉行的包裝術：先把可遷移技能（產品運營、數據分析、KYC 流程、BD 談判）以成果化語言呈現，做小而可驗證的 Web3 作品集：參與 DAO、完成 bounties、在測試網部署簡易合約或做社群活動策劃並保存案例；撰寫 1–3 篇深度學習筆記或案例分析貼到個人領英/Medium，並在社群（Discord/TG）主動貢獻以建立聲譽。面試時以「我如何用既有技能解決鏈上問題」作為主線，展示快速學習能力與實戰樣本比單純宣稱學過課程更具說服力。

遠程辦公優劣：優點是地理與時間彈性、可接全球職缺、節省通勤成本；但缺點包括時差協調、溝通成本高、團隊凝聚力較弱、資安風險與合規（跨國薪資、社保、稅務）更複雜。建議遠程團隊建立清晰的 async 溝通規範（文檔化決策）、固定同步節奏（週會）、明確 KPI 與任務追蹤工具，並為員工提供資安指導（硬體錢包、雙因素驗證）。

2. 即時互動與客服——在 Discord/Telegram 回應使用者問題、引導新手、處理投訴與投票疑問；

3. 社群氛圍管理——制定並執行社群守則、移除惡意帳號、化解爭議；

4. 活動與激勵運營——設計任務、空投、貢獻者計畫並追蹤白名單與防刷策略；

5. 數據監測與洞察——監控 DAU、訊息量、提案參與率、留存與鏈上指標，做出內容與激勵優化。

社區維護要點在於「制度化」與「可量化」：
建立明確的上線流程、新手引導（歡迎包、FAQ）、貢獻者等級制度與治理流程，並用儀表板追蹤核心 KPI；同時要做到透明與可驗證——公開公告決策紀錄、治理提案結果與空投條件，降低誤解與不信任。風控與合規不可忽視：主動教育成員不要簽署可疑合約、統一發布官方連結、對高風險活動（空投、募資）做 KYC/審計前置。人才與角色分配方面，應有社群管理員、內容編輯、數據分析與安全專員分工協作，並透過定期內部同步與回饋機制快速迭代。

最後，社群是長期資產：把每次互動視為信任累積的契機，透過故事化內容、真實案例分享與公開回饋機制，把被動成員轉化為活躍貢獻者，才能在去中心化生態中建立持久競爭力。

# 2025-08-14

今日研究主題：Twitter Space 與其他社區聯動的運營要點

1. 運營要點
- 明確合作目標：教育／導流／募資／治理，雙方需事前一致。
- 受眾與內容對齊：確認目標受眾輪廓與主題深度，避免內容兩頭不到岸。
- 曝光與宣傳節奏：T-7/T-3/T-1/當日倒數，多渠道同步（X、Discord、TG）。
- 嘉賓與分工書面化：嘉賓名單、主持與議程分工、發言時長事先確認。
- 獎勵與防刷規則：若有空投/Whitelist，事前定義白名單、KYC、反刷機制。
- 技術彩排：雙方共同演練音訊、連線、錄音與回放授權。
- 互動機制設計：預收問題＋現場 upvote 決定順序；安排社群管理員協助篩選。
- KPI 與數據共享：事前同意衡量指標（peak listeners、互動數、社群新增）與資料交換方式。
- 內容沉澱計畫：會後共同產出短影音、Q&A、摘錄與回放鏈接。
- 合作契約/備忘錄：必要時以簡短 MOU 固化權責與贊助條款。

2. 技術與流程（執行面）
- 指定主機與備援：主主持端＋備用連線、備用麥克風、替代講者。
- 音訊品質檢查：避免回音、延遲，確認每位講者麥克風音量。
- 錄製與授權：明確告知錄音/回放政策並取得同意。
- 時區考量：選擇對雙方社群友好的時間，並提供回放。

3. 互動與社群管理
- 問題池管理：有人負責收集、篩選與 upvote，確保高質量問題優先。
- 主持人節奏控制：控制討論長度、避免嘉賓互相打斷或話題偏離。
- 防刷與審核：監控機器人刷票、冒充賬號和釣魚鏈接。

4. 合規與安全
- 風險揭示：涉及空投或投資相關內容，應有風險聲明與合規提示。 
- 官方鏈接驗證：只發佈官方 URL / QR，避免第三方短連結。
- 不做投資建議：公開場合避免具體投資推薦或保證收益。

# 2025-08-12

本日學習主題：Web3線下活動（策劃 → 籌備 → 執行 → 復盤）

1. 策劃：先明確目的（教育、產品發佈、招募貢獻者或募資）、目標受眾（開發者、投資者、社群用戶）與 KPI（出席率、留資率、社群成長、內容轉化）。選定形式（workshop、hackathon、論壇、MeetUp）、時間與預算，並列出風險清單（跨境講者、合規敏感議題、現場安全）。設計報名機制時可考慮鏈上白名單或 NFT 門票，但要注意個資保護與法務合規。

2. 籌備：場地與網路為重中之重──確認場地容納人數、備援網路、投影與音響設備、充電站與硬體錢包存放點。提前溝通講者（brief、講稿、時長限制）並排定彩排。準備物料：簽到台、QR code 簽到、會場導引、法律免責與拍攝授權。財務與贊助需明確合約條款與權益。風控面要安排安保、現場客服、以及展示合約經過審計或 demo 的安全證明。

3. 執行：依議程嚴格控場，主持人把控節奏並處理超時。簽到流程要快速且具備身份或錢包校驗（視活動需求）。技術演示備有回退方案（模擬帳戶、備援私鑰）。互動設計（現場問答、投票、即時空投）需防刷與白名單機制。全程記錄（錄影、直播、截圖），並按預案處理突發事件（講者缺席、網路中斷或安全事件），同時以透明方式向參與者溝通狀況。

4. 復盤：活動後 48–72 小時內彙整關鍵數據（到場率、留資率、社群新增、回放觀看數、NPS）。沉澱內容：剪輯短片、整理問答集、上傳投影片並在社群分享。進行風險與安全復盤：是否發生詐騙、釣魚或技術漏洞，處理流程與結果為何。完成財務結算與贊助回報，並根據 KPI 與參與者回饋形成改進清單（議題深度、互動形式、彩排次數）。將學到的經驗寫成可重用的活動 Playbook，便於未來複製與擴展。

# 2025-08-10

今日學習主題：Web3釣魚詐騙等風險與防範意識

在 Web3 生態中，釣魚詐騙與社交工程手法日益多元：常見類型包括偽造官網或錢包授權頁面引導簽名、假空投／假客服誘導簽署惡意合約、惡意合約透過過度批准（approve）竊取代幣、以及項目方或流動性池的 rug-pull。這類風險被放大是因為鏈上交易不可逆、帳戶具假名性且跨境特性使追索困難。

防範重點在於「習慣」與「技術」雙管齊下：使用者端應養成錢包衛生（只點官方連結、使用硬體錢包簽署重要交易）、在授權代幣時限制數額或使用授權管理工具撤銷不必要的 approve、不隨意簽署陌生訊息或合約呼叫，並安裝防釣魚擴充與檢查域名細節；遇到疑問則先在官方頻道驗證而非直接回覆私人訊息。

項目方與平台層面則應做到透明化（公開合約地址、審計報告）、建立可驗證的官方通訊標識、提供明確客服與緊急應變流程、設置 bug bounty 與鏈上監控（異常交易警報、地址風險分級）。同時透過社群教育（案例分享、操作教學、模擬演練）提高整體防詐意識。結論是：在不可逆的鏈上世界，預防勝於事後補救，技術防護與持續教育缺一不可。

# 2025-08-09

今日研究內容：Web3薪酬結構與僱傭型態的法律風險 

在 Web3 薪酬結構與僱傭型態上，法律風險呈現多面向且具跨境複雜性。

首先，代幣薪酬（token wages / token grants）可能觸及證券法或投資產品定義：若代幣被視為證券，發行與分配需遵循公開發行、登記或豁免規則，否則企業與受領者都面臨刑事或民事責任。

其次，代幣波動與可變現性造成薪酬價值不穩，涉及薪資支付法規、最低薪資與勞動報酬保障問題；若以代幣部分代替法幣支付，企業需確認當地勞動法是否允許，並妥善處理稅務與社保代扣代繳義務。

再者，DAO 與去中心化協作常模糊「員工」與「承包商」界線，導致勞動關係判定、雇主責任與雇傭保障難以落定；跨境分散的貢獻者亦使適用法域與管轄權不明。

合約設計上，須注意代幣解鎖期（vesting）、回收（clawback）條款與可執行性，以及就知識產權、保密與競業限制的歸屬與約束力。從合規角度，還要考慮反洗錢與 KYC 規定、薪酬涉及穩定幣或虛擬資產時的資金流監管，以及個資保護法對薪資資料的要求。

建議實務作法：

一、在發放前由法律顧問進行證券屬性與稅務風險評估；

二、以書面合約明確薪酬組成、法幣折算機制、vesting 與解除條件；

三、採用合規的支付與稅務服務商處理跨境薪酬與代扣；

四、針對 DAO 型組織設計清晰治理與雇傭政策，並在必要時採用受監管實體作為支付方；

五、持續關注各司法區法規動態，將法規風險納入產品與人力策略的常態化評估。

只有把合規嵌入薪酬設計與運營流程，Web3 的薪酬創新才能在法律框架下穩健推進。

3. 擬定腳本與 FAQ：準備開場介紹、主要問答框架與預先收集的問題；製作常見問題清單供講者參考。

4. 宣傳節奏：發布活動海報、倒數貼文、關鍵時間點提醒（T-7、T-3、T-1 天），並在 Telegram/Discord/X/Medium 同步推廣。

5. 技術與流程彩排：確認直播工具、音訊/視訊、授權發言者、備用連線，進行至少一次完整彩排。

6. 主持與引導流程：指定主持人負責節奏控制、篩選問題、維持討論聚焦；同時安排社群管理員負責即時答疑與鏈接資源。

7. 互動機制設計：設立提問入口（表單、訊息池），運用投票或 upvote 機制決定優先問題；可設有小獎勵（NFT、空投 whitelist）提升參與率。

8. KPI 監測：追蹤參與人數、發問數、平均停留時長、推文互動、錄播回放觀看次數與後續社群成長。

9. 會後整理與延伸：將 AMA 精華整理成問答集、剪輯短影音、發布會後報告，並把未回答問題列入下一次議題或 FAQ。

10. 回饋與優化：蒐集參與者回饋問卷，分析數據並在下次活動中執行改進（時間、主題深度、互動設計）。

# 2025-08-07

今日研讀了Web3與反洗前有關的相關內容：

在 Web3 生態中，去中心化與匿名性的設計雖為用戶隱私提供保障，卻也為洗錢者創造了溫床。首先，跨鏈橋（bridge）與去中心化交易平台（DEX）可在不同公鏈之間迅速轉移資金，且無需 KYC 認證，使資金來源更難追蹤。其次，混幣協議（如 Tornado Cash）透過切割與混合交易，打破資金流向的可追溯性，加劇監管機構的偵查難度。此外，NFT 市場的高價交易亦能成為洗錢工具，以藝術品定價掩護非法資金注入。面對這些挑戰，鏈上風控需結合鏈下監管，採用地址風險分級、行為模式分析與 AI 檢測可疑交易；同時推動去中心化身份（DID）與零知識證明（zk-KYC）等技術，實現隱私保護與合規要求的平衡。在未來，Web3 生態必須加強協作：項目方要主動落實 AML/CFT（反洗錢／打擊資恐）流程，錢包與橋接服務商需內建風險篩查模組，監管部門應更新法律框架，以鏈上數據為基礎，確保技術創新不被不法利用。

釣魚簡介會議前有興趣的內容預習與筆記：
全球 Web3 合規與法律風險總覽
1. 監管模式多元化：各國對數位資產的分類不同，美國SEC將大部分ICO代幣視為證券，CFTC則將加密資產定位為商品，FinCEN認定其為資金轉移工具。因而，Web3 服務供應商（交易所、錢包、託管等VASP）通常需取得執照，並遵守類似傳統金融的KYC/AML要求。FATF「旅行規則」要求交易所等共享用戶交易資訊，全球已有73%司法管轄區立法實施或正推動此規則。
2. 證券法與金融法規：若虛擬資產被視為「投資契約」，則需符合證券法（如1953年證券法）。許多司法區採用「豪伊測試」（Howey Test）判斷是否存在投資契約；如以募資或預期未來收益為目的發售代幣，可能屬證券發行。此外，某些穩定幣若掛鉤法定貨幣，也可能適用電子貨幣或支付服務法規。
3. 穩定幣監管：穩定幣被全球監管者列為重點，焦點為儲備資產與兌換機制。歐盟MiCA法規對單一貨幣穩定幣（EMT/ART）提出嚴格規定，如必須全額兌換、發布白皮書、銀行級儲備披露等。新加坡MAS也要求法幣掛鉤穩定幣具備高流動性儲備、資本要求和清算機制。美國近期通過《GENIUS法案》，以推動科技企業發行穩定幣同時加強監管。全球各地的實體或擬議監管均強調：“1比1”儲備兌換與強化審計披露，以防範資產荒、流動性枯竭風險。
4. 跨境合規與協調：Web3本質跨境，監管亦向統一標準靠攏。FATF等國際組織要求成員實施VASP許可制度及AML/CFT規範，並警示不能「全盤禁止」避免無法監管。2025年G7/G20陸續提出政策建議（如白宮數位資產報告），鼓勵國際協作，諸如共同打擊洗錢、防範金融犯罪等。各地監管沙盒、共同認證框架等趨勢可望降低企業跨境合規成本。
